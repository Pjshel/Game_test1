# 局内系统定义

版本 v0.1-final(生效) | 宪法集成员
修订轨迹:v0.1草案 → v0.2(五行元素方案)→ 废弃回滚 → **v0.1-final**(三项拍板并入:滚雪球放开、元素定案火/冰/电、因子内容后置)
核心思想:**我们不设计四百把武器,我们设计一套能长出四百把武器的语法。**

---

## 1. 总纲:语法封闭,数据开放

四套系统共用同一范式:**每套系统 = 一张封闭的"词汇表" + 一条开放的"数据表"**。词汇表(发射模式、弹体行为、元素效果、天赋钩子)是代码,扩充需升版审批;数据表(具体武器、具体天赋)是JSON,内容阶段自由增删。语法封闭保证架构不漂移,数据开放保证内容能量产——承重墙"内容即数据"在设计层的对应物。

## 2. 武器系统【跟随】

**组合语法**:一把武器 = 大类 × 发射模式 × 弹体行为 × 命中效果 × 能量档位 × 稀有度 × 视觉签名。

三张词汇表(**加粗为MVP切片启用范围**):

- **发射模式(7种,封闭集)**:**点射、连射、扇形、挥砍**、蓄力、射线、抛射。MVP完整版目标15把武器覆盖全部7种模式,每模式至少2把——刚好压测语法完备性。
- **弹体行为**:**直线、反弹(N次)、穿透(N层)、分裂(触发条件+子弹数)**;追踪、回旋、环绕后置。
- **命中效果**:**击退、元素附加、爆炸(半径AoE)**;连锁牵引类后置。

**能量经济三定律**:

1. **效率与功率负相关(结构决策)**:同稀有度内,瞬时DPS每上一档,伤害/蓝效率必须下一档——高功率武器天生是"Boss战掏出来的",此定律是换枪节奏的数学来源;
2. **零耗保底**(宪法既有):蓝耗档位 0耗 / 轻(1) / 中(2~~3) / 重(4~~6) / 特殊(按tick或按蓄力);
3. **进攻产能量**:能量球主要挂在击杀事件上掉落,躲着不打=能量枯竭——经济系统本身奖励进攻(此条喂给WP5掉落表)。

**稀有度=机制阶梯(结构决策,承接动机定义"颜色≠强度")**:白=语法基础词,单一机制,承担教学;绿/蓝=一个变化词(如反弹);紫/橙=两词组合(如分裂+元素);红=破格词,允许携带独有规则。配套防线:**红武DPS上限 ≤ 白武 × 2.5**(v0参数,防数值膨胀吃掉机制设计);登场感预算(VFX/音效)随稀有度递增而伤害不递增——炫酷花在表现上,不花在数字上。

**近战的系统身份**:近战普遍0耗或极低耗,叠加宪法近战消弹判定——近战=能量见底时的防御性保底,在经济系统里有明确岗位。

数值锚点沿用《核心体验定义》§5的TTK基准,本文档不另起数值体系。

## 3. 元素与状态【跟随 · 已拍板:火/冰/电】

**触发律(结构决策,照搬原版最妙的一条)**:元素武器**暴击不造成双倍伤害,改为必定触发元素**;非暴击小概率触发(v0=10%)。元素频率由此被暴击率天然限流,速射武器永冻Boss这类问题在规则层面直接不存在。

**MVP三元素(已定案)**:

- **火** = 灼烧DoT,1伤/0.5秒 × 4跳,同源刷新不叠层——刷新族;
- **冰** = 冻结定身1.2秒,Boss受控时长 × 0.3(反无限控)——硬控族;
- **电** = 连锁,向2格内最近敌人跳跃,传递50%伤害,至多2跳——群体族,三者里最上镜,服务"炫酷"验收。

毒(叠层DoT)与眩晕后置。

**状态叠加策略词汇表**:刷新 / 叠层(带上限) / 免疫窗口——三选一写进每个状态的数据行。

元素之间互不作用(与原版一致);元素关系系统已归档至创新储备(见CONSTITUTION.md §4)。

## 4. 天赋系统【跟随 · 已拍板:滚雪球放开】

**两物种与行为红线(结构决策)**:天赋分"改数字"(修饰器客户:暴击+15%)与"改行为"(事件钩子客户:碎盾时释放消弹冲击波、击杀10%概率回血)两个物种,**行为型占比 ≥ 40%**——天赋的职责是改变本局决策,不是加法器。

四系分池:火力 / 生存 / 资源 / 异化(规则改变者),池权重 v0 = 30/25/20/25%。

**获取与重掷**:每清一层楼三选一;槽位上限=楼层数(具体数字待单局结构层定稿);重掷沿用奸商经济,金币起价、逐次翻倍;抽取规则 = 加权池 − 已拥有 − 互斥标签;**Build保底**:连续两层未出现玩家主轴类别时该类权重翻倍——有界随机替玩家的Build连贯性兜底。

**滚雪球哲学(结构决策⑧,已拍板:放开)**:允许乘区叠乘产生质变,**不设全局硬上限**,平衡手段只用掉率与互斥标签。理由:乘区滚雪球是品类爽点本体("CD减半+冷却缩减=无限炮台"式高光的来源);传奇档成就需要系统存在"造神上限"才有炫耀素材;保守封顶省下的平衡工作,会以"Build全都差不多"的乏味加倍偿还。

**角色被动 = 天赋系统的常驻实例(结构决策)**:同一套机制,不为角色被动另开系统——架构复用即设计纪律。

## 5. 挑战因子【跟随 · 已拍板:内容后置】

**定位**:玩家自选的难度调制器 + 未来每日种子的变化源。三池刻意偏负(v0 = 负45% / 正25% / 混合30%)——偏负才有"逆风翻盘"的叙事价值。

**难度预算算法(结构决策,"有界随机"的落地)**:每个因子携带难度分(−3~+3),组合抽取后求和,总分落在玩家所选挑战档的目标带宽外则整组废弃重掷(例:"标准挑战"带宽 [−2, +1])。

**双轨作用面**:数字类因子走修饰器;规则类因子(禁宠物、楼层乱序、商店涨价)走WP2的规则标志注册表。

**范围(已拍板)**:钩子随WP2/WP6进切片;**因子内容整体后置到MVP完整版**——切片阶段验证的是管道,不是难度选项。

## 6. 本文档不做什么

不设计具体武器与天赋清单(内容生产阶段);不含敌人、房间、Boss与金币经济(下一层:单局结构定义);不做精细数值平衡(灰盒与试玩阶段)。

## 7. 对下游工作包的连锁修订

- **WP3**:武器schema字段就此冻结——三张词汇表的枚举 + 稀有度 + 视觉签名 + 能量档位 + element枚举(火/冰/电)。
- **WP4**:元素表直接按§3落成数据;交付物=注册表+三状态+叠加策略。
- **WP6**:天赋池schema、Build保底、重掷经济。
- **WP2**:规则标志注册表为一等公民。
- **WP5**:敌人schema保留可空element字段(零成本保险,不承诺任何系统)。
- **内容生产纪律**:未来所有批量生成的武器/天赋,必须只使用已批准的词汇表组合——zod在CI里执法。

## 8. 变更规则

**结构决策(八条)**:①三张词汇表为封闭语法 ②效率功率负相关律 ③进攻产能量 ④稀有度=机制阶梯 ⑤元素触发暴击律 ⑥行为型天赋≥40% ⑦因子难度预算制 ⑧滚雪球放开,无全局硬上限。改动任何一条需升版并由架构轨评估连锁影响。

**v0参数**(一切带数字的):灰盒与试玩阶段自由调整。
