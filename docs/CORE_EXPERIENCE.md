# 核心体验定义

版本 v0.1(暂定生效) | 宪法集成员
参数说明:一切带数字者均为 v0 调参起点,灰盒定稿后回填升版 v1.0。
标注:【跟随】=沿用元气骑士的结构选择;【适配】=因浏览器平台/项目范围做的调整。

---

## 1. 一句话体验目标【跟随】

> **"注意力永远在脚下,而不是准星上。"**
> 一局二十分钟的走位爽游:瞄准交给锁定,火力交给能量,生死交给走位,成长交给Build。

三个"不是",划清边界:

- **不是**手动瞄准的技巧射击(那是挺进地牢);
- **不是**弹幕地狱STG(压力来自弹幕的量与角度,不来自单弹速度);
- **不是**大数值RPG(拒绝前传式面板,伤害永远是个位数小整数)。

## 2. 瞄准方案【跟随 · 本文档最高优先级结构决策】

**自动索敌。MVP不做任何形式的手动瞄准,连可选项都不做。**

规则定义:

- 锁定"有视线判定的最近敌人"(墙后不锁,模拟层做射线检测);
- 锁定目标头顶显示小标记(可读性);
- 无目标时朝移动朝向射击;
- 目标切换仅由距离变化自然发生,不设手动切换键;
- 鼠标在本游戏里**只是另一个开火键**,坐标不参与瞄准。

理由(为何是身份而非设置):自动索敌把100%的技巧表达转移到走位上,敌人设计随之全部围绕"弹幕图案与移动压迫"而非"难以命中";键盘、触屏、手柄共享同一套操作模型(浏览器游戏的输入平权红利);双瞄准方案等于同时平衡两款游戏,永久排除。

## 3. 核心动词表【跟随】

五个动词,一个都不多:

1. **移动**(WASD / 虚拟摇杆)
2. **开火**(按住持续攻击,消耗能量)
3. **换武器**(2槽位,即时切换)
4. **技能**(角色专属,CD制)
5. **交互**(拾取/开门/NPC)

明确排除的动词(结构决策):**通用翻滚/冲刺**(带无敌帧的位移会彻底改写弹幕密度逻辑;位移能力保留给未来角色技能,守住基线)、跳跃、格挡键、手动瞄准。

两条附加结构规则:

- **开火即决策**:能量是资源,开火必须是玩家主动持续按住的选择,拒绝吸血鬼幸存者式全自动(这是能量系统有意义的前提);
- **近战消弹**:近战挥砍判定内销毁敌弹,这是近战的防御性身份,归入开火动词的武器特性;MVP近战武器即带此规则,弹反留给后续天赋。

## 4. 生存模型【跟随】

**结构规则**:

- 盾+血双层,外加能量,构成三层压力节奏:能量管当下,盾管短线失误,血管整局资源;
- 受击优先扣盾;盾可自动回复,**血绝不自然恢复**(只靠药水与房间奖励);
- 受击有无敌帧,期间子弹穿过角色;
- 伤害与血量全部小整数;
- **永远存在零耗保底输出**:初始武器蓝耗为0,能量见底永远不软锁。

**v0参数**(灰盒调):基准角色 5血/4盾;普通敌弹伤害1、精英与Boss伤害2;脱离受击3秒后每秒回1盾;无敌帧0.6秒;能量池200;蓝耗档位 0耗 / 轻(1) / 中(2~~3) / 重(4~~6) / 特殊(按tick或按蓄力)。

## 5. 节奏基准【适配 · 全部为v0参数】

- **移速与弹速**:玩家横穿标准房间(16格宽)约3秒;敌弹基准速度 ≈ 玩家移速 × 1.1~~1.3——单弹永远躲得掉,压力靠数量与夹角制造;中距离(6格)交战反应窗口目标 0.7~~1.0 秒。
- **TTK**:基准武器 2~~3 发/秒;普通小怪 2~~4 发命中死,精英 8~~12 发;单个战斗房 20~~40 秒清完,战斗与拾取决策交替形成"紧张30秒—释放15秒"循环。
- **弹幕密度档位**:同屏敌弹 常态≤15 / 高压≤40 / Boss阶段≤80(对照WP3的500弹性能预算,余量充足)。
- **时长**:完整版一局 15~~25 分钟;MVP垂直切片一层楼 3~~5 分钟。

## 6. 手感参照物【适配】

**正参照**:整体节奏与索敌手感 = 元气骑士原版(唯一全局对标);打击感与juice = Nuclear Throne(顿帧、震屏、尸体物理);弹幕可读性与预警设计 = 挺进地牢(**仅**参照其telegraphs,不参照其操作)。

**反参照**:挺进地牢的翻滚+手瞄;吸血鬼幸存者的零决策自动开火;元气骑士前传的大数值面板。

**Juice验收清单**(合计不到一天工时,却是"手感好"的全部来源):

- 命中顿帧 30~50ms(击杀取上限);
- 受击闪白 2 帧;
- 屏幕震动三档(命中微 / 爆炸中 / Boss大);
- 敌人受击击退 0.3~0.5 格;
- 弹壳与命中粒子;
- 音效三层(开火/命中/击杀)。

## 7. 灰盒验收方式

一个空房间、色块角色、一把基准枪、三种靶子(静止/游走/会还击),零美术、零生成、零天赋。

**唯一合格线:产品负责人打靶五分钟后还想继续打。**

施工硬性要求:第4、5、6节所有带数字的参数做成**局内调试面板**(滑杆实时生效)——现场调、现场拍板,不碰代码;定稿数值回填本文档升级为 v1.0。

## 8. 变更规则

**结构决策(七条)**:自动索敌、无通用翻滚、盾血双层+血不自愈、小整数数值、开火即决策、零耗保底、参照系。改动任何一条需升级文档版本,并由架构轨重新评估对下游WP的连锁影响。

**v0参数**(一切带数字的):灰盒阶段自由调整,无需审批。
