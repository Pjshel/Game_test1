# 美术素材白名单(操作文档)

状态:生效 | 层级:操作文档(不入宪法,受宪法附则A管辖)
维护:架构轨 | 执行:Claude Code | 核验日期:2026-07-09(经Tavily检索并核验许可页原文)

## §1 许可三级制

- **绿区(默认可用)**:CC0 / 公有领域。零义务,可修改、可商用、可随游戏发布。仍须登记CREDITS(礼貌性,非法律义务)。
- **黄区(可用,带义务)**:CC-BY 4.0 及同类署名许可;OFL字体许可。义务:登记CREDITS并进入游戏内致谢界面;OFL字体须随发行保留许可文本,且不得单独售卖字体本身。
- **红区(一律拒绝)**:CC-BY-NC及一切"非商用"许可(锁死未来商业化);"仅限个人使用/免费但看情况"类模糊授权;GPL类传染性许可素材;**任何从商业游戏提取、截取、重绘的素材与同人素材——《元气骑士》相关素材为绝对禁区,零例外**。

**核验程序**:以素材页面的许可声明为唯一依据(itch.io 看页面 "Asset license" 字段;个人站看作者原文)。导入时将许可声明原文摘录进CREDITS登记项。页面无明确许可声明 = 视为红区。

## §2 风格与技术匹配规则

1. **附则A过滤器**:现代/科幻像素风;拒绝国风古风;拒绝中世纪奇幻角色(环境砖可酌情筛用)。
2. **主包定基调**:全项目认定一个主素材包确定分辨率(16×16 或 32×32,二选一)与色板基调,其余包只做补充并须缩放/调色对齐——混搭撕裂是占位美术最常见的翻车点。
3. 占位素材同样受本规则约束(宪法附则A明文覆盖调试UI与占位素材)。

## §3 已核验白名单

### 3.1 综合基地 · Kenney(kenney.nl)
- 许可:**全站CC0**,可商用,无署名义务(官网与社区多方确认)。
- 直接对口的包:**Top-down Shooter**(俯视角角色/枪械/砖块)、**Pixel Shmup**(像素弹幕机)、**1-bit Pack**(肉鸽常用极简风)、**UI Pack系列**(现代扁平UI,完美匹配附则A)、音效包系列、字体包。
- 亦有 All-in-1 打包(60,000+素材单次下载,含2D/音频/字体)。
- 定位:**默认主基地**;UI与音效的第一供给源。

### 3.2 风格最贴合 · Sci-Fi Facility Asset Pack(作者 MurphysDad,itch.io)
- 许可:**CC0**(页面 Asset license 字段明示 Creative Commons Zero v1.0 Universal,作者注明署名可选)。
- 内容:俯视角科幻像素——双角色四向移动精灵、设施墙/地砖、动画物件(计算机、电闸、发光球、按钮)、装饰件。
- 定位:**风格样板与角色/场景占位首选**——俯视角+科幻+像素三条件全中。

### 3.3 地牢环境 · 0x72 两代Dungeon Tileset(0x72.itch.io)
- 许可:**CC0**(作者页面原文:可用于私人与商业项目、可修改、无署名义务)。
- 内容:16×16地牢tileset两代(II代含重绘与扩充),肉鸽圈最广泛使用的占位tileset。
- **风格警告**:整体偏中世纪奇幻——环境砖(墙/地/火把)可筛用,骑士/骷髅类角色**弃用**(违反附则A)。

### 3.4 检索活水入口(许可预过滤,后续找素材从这里进)
- itch.io CC0过滤 · Top-Down标签:https://itch.io/game-assets/assets-cc0/tag-top-down(核验时184个结果)
- itch.io CC0过滤 · Dungeon Crawler标签:https://itch.io/game-assets/assets-cc0/tag-dungeon-crawler
- itch.io 免费 · Sci-fi+Top-Down交叉标签:https://itch.io/game-assets/free/tag-science-fiction/tag-top-down(此入口未预过滤许可,逐项核验)
- OpenGameArt.org:按CC0过滤检索,Kenney作品在此亦有镜像合集。

### 3.5 字体
- **Press Start 2P**(设计者 Codeman38):**SIL OFL 1.1**,可商用;经Google Fonts引入网页最省事。义务见§1黄区。
- 备选:Google Fonts 内其他OFL像素字体(如 VT323),同一许可框架。
- 中文像素字体留待包装层单独立项核验(中文字体许可雷区更多,届时专项处理)。

### 3.6 音效
- **灰盒期(现行纪律)**:Web Audio 合成自制(WP1.5提示词已规定)——零许可风险,且参数可进调试面板。
- **升级期**:Kenney 音效包系列(CC0);浏览器端 sfxr 类生成器自制音效(自制=自有版权)。

## §4 CREDITS登记纪律

- 仓库根建立 **CREDITS.md**;每导入一项素材登记一行:素材名 / 作者 / 许可(附页面许可声明摘录)/ 来源URL / 导入日期 / 用于何处。
- 黄区素材额外进入游戏内致谢界面(界面本体属包装层,数据从CREDITS.md生成)。
- 无登记不导入——CI阶段可由Claude Code做一致性抽查(assets目录文件 ↔ CREDITS条目)。

## §5 导入时机

1. **WP1.5(灰盒)**:维持宪法规定,**零美术素材**,纯几何色块——本白名单在该阶段不生效。
2. **灰盒手感定稿后**:素材作为"占位升级"入场,主包定基调(§2)。
3. **注意**:CC0素材在法律上可直接随正式版发布——它们不只是占位,在原创美术(包装层)完成前可合法充当正式素材。原创像素美术仍是包装层的既定目标。

## §6 给Claude Code的执行规则

导入任何素材前:核验许可(§1程序)→ 确认风格匹配(§2)→ 登记CREDITS(§4)→ 方可入库提交。白名单外的新来源:仅当核验结果落在绿/黄区且完成登记后方可使用,并回报架构轨将其增补进本文档。红区素材在任何情况下不得进入仓库,包括"临时测试"。
